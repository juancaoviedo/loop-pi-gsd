## Phase Goal

**As a** Flow operator running `/gsd-spec-phase` and `/gsd-discuss-phase`, **I want to** have interactive questions answered by an independent Agent B running in a separate OS process with its own provider/model, **so that** delegated ask/answer is genuinely isolated, fail-closed, and evidence-backed.

<objective>
Harden the isolated runtime slice from Plan 01 into a fail-closed, provider-isolated one: load Agent B strictly from its own config path, exercise a provider/model distinct from Agent A, and normalize every transport/protocol/timeout/crash failure into bounded retries that end in `escalate-and-block` with stable machine-readable reason codes.

Purpose: The runtime boundary is only trustworthy once startup, provider separation, and the full failure matrix are deterministic and non-fallback.
Output: `agent-b-config.mjs` (dedicated fail-closed config loader), `agent-b-reason-codes.mjs` (namespaced reason-code catalog), failure-policy wiring in `agent-b-runtime.mjs`, provider-echo wiring in `agent-b-entry.mjs`, and a passing transport/failure test.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| filesystem config file → runtime | Agent B config may contain secrets; only allowlisted keys may cross into runtime state and evidence |
| Agent B failure signals → escalation decision | Timeouts, crashes, and malformed payloads must deterministically map to reason codes, never to a permissive continue |
| startup failure → control flow | A failed separate-process startup must halt, never reroute to in-process/same-provider answering |

## STRIDE Threat Register (ASVS Level 1; block on high severity)

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-09-05 | Spoofing/Elevation | startAgentB startup gate | high | mitigate | On config/startup failure, surface a startup reason code and take no in-process answer path (no silent fallback, P4) |
| T-09-06 | Information Disclosure | agent-b-config.mjs loader | high | mitigate | `AGENT_B_CONFIG_ALLOWLIST` strips non-allowlisted keys so secrets never reach runtime state or evidence |
| T-09-07 | Denial of Service | per-try timeout in agent-b-runtime.mjs | high | mitigate | `AbortSignal.timeout` bounds each attempt; retry cap 2 prevents unbounded loops; exhaustion escalates-and-blocks |
| T-09-08 | Tampering | failure classification | medium | mitigate | `classifyAgentBFailure` maps all failure classes to stable namespaced codes; unknown protocol errors default to a retryable malformed code |
</threat_model>
