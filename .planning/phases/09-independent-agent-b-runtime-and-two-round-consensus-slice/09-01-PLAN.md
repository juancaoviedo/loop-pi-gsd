## Phase Goal

**As a** Flow operator running `/gsd-spec-phase` and `/gsd-discuss-phase`, **I want to** have interactive questions answered by an independent Agent B running in a separate OS process with its own provider/model, **so that** delegated ask/answer is genuinely isolated, fail-closed, and evidence-backed.

> **ROADMAP wording note:** The ROADMAP `**Goal:**` line still references "two-round consensus." The locked `09-SPEC.md` scope (authoritative per CONTEXT.md spec_lock) descopes consensus/debate to future phases and delivers real independent ask/answer runtime only. This plan set follows the SPEC. Consider running `/gsd mvp-phase 9` to align the ROADMAP goal wording.

<objective>
Deliver the thinnest real end-to-end vertical slice: spawn Agent B as a genuine separate OS process, complete a startup handshake, and run one `/gsd-spec-phase` delegated ask/answer round over a strict stdio JSONL envelope, persisting sanitized runtime evidence.

Purpose: Prove the isolated process boundary and protocol contract exist and work for the happy path before layering config isolation, failure policy, and the second workflow surface.
Output: `agent-b-protocol.mjs` (JSONL envelope contract), `agent-b-entry.mjs` (child runtime harness / second gsd-pi instance boundary), `agent-b-runtime.mjs` (parent process manager), and a passing runtime integration test.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| parent orchestrator → spawned Agent B process | Command/args cross the OS process boundary; must not be shell-interpolated with untrusted input |
| Agent B stdout → parent JSONL parser | Untrusted line-delimited payloads cross back into the parent and must be validated fail-closed |
| runtime → run-local evidence files | Provider metadata written to disk must be sanitized to allowlist only |

## STRIDE Threat Register (ASVS Level 1; block on high severity)

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-09-01 | Tampering/Elevation | spawn args in agent-b-runtime.mjs | high | mitigate | Use `spawn(process.execPath, [entryPath], ...)` with no `shell: true`; never build a shell command string from question/provider input |
| T-09-02 | Tampering | JSONL ingestion in agent-b-protocol.mjs | high | mitigate | `parseAgentBLine`/`validateAgentBEnvelope` fail closed; malformed lines become retryable protocol errors, never permissive parsing |
| T-09-03 | Information Disclosure | evidence writer in agent-b-runtime.mjs | high | mitigate | Route all provider metadata through `sanitizeProviderMetadata`; persist only the free-text answer + allowlisted metadata |
| T-09-04 | Denial of Service | child lifecycle in agent-b-runtime.mjs | medium | mitigate | `stopAgentB` graceful-then-force kill with bounded timeout; test asserts close/exit fires |
</threat_model>
